MOUND — SCRIPT BREAKDOWN
How the scripts work together
================================================================================

OVERVIEW

Every 30 minutes, cron fires three scripts simultaneously:

    log_sensors.py      — gathers data from all sensors
    update_site.py      — generates graphs and publishes to website
    camera.py           — captures noon photo once daily

These three scripts are completely independent of each other.
If one fails, the others keep running.

Additional utility scripts (run manually as needed):

    graph_day.py        — generate/regenerate a graph for any specific date
    generate_7day.py    — force-regenerate the 7 day graph immediately

================================================================================

log_sensors.py
PURPOSE: Read all sensors and append one line to the monthly data file.

WHAT IT DOES:
1. Sets up I2C bus
2. Reads each sensor individually, each in its own try/except block
   - BME280  -> temp, humidity, pressure (I2C 0x76)
   - TSL2591 -> lux (returns 99999 on overflow instead of crashing) (I2C 0x29)
   - ADS1115 -> soil moisture (analog channel A0) (I2C 0x48)
   - DS18B20 -> soil temperature (reads from /sys/bus/w1/devices/)
   - GPIO17  -> rain digital (0=dry, 1=rain)
   - INA219  -> voltage and current draw (I2C 0x40)
3. If a sensor fails, logs the error to errors.txt and uses None for that value
4. Appends one CSV line to ~/mound/data/YYYY-MM.txt
5. Creates the monthly file with a header comment if it doesn't exist yet

DATA FORMAT:
# timestamp, temp_f, humidity_pct, pressure_hpa, lux, soil_temp_f, soil_moisture_raw, rain, voltage, current_ma
2026-06-01 00:00:02, 62.0, 59.5, 988.4, 123.2, 69.3, 16122, 0, 5.08, 212.4

KEY BEHAVIOR:
If any individual sensor fails, the script keeps going and logs
whatever it can. MOUND never goes fully silent due to one bad sensor.

================================================================================

update_site.py
PURPOSE: Generate graphs and publish the data page to botpi/website.

WHAT IT DOES (every 30 minutes):
1. Waits 10 seconds for log_sensors.py to finish writing
2. Writes a DEBUG line to errors.txt with current hour/minute
3. Calls generate_graphs.py to produce today.png with dynamic ranges
4. SCPs today.png to botpi:/var/www/html/mound/graphs/
5. Calls generate_page.py to produce index.html and latest.json
6. SCPs index.html to botpi:/var/www/html/mound/
7. SCPs latest.json to botpi:/var/www/html/mound/data/

WHAT IT DOES (at 23:30 only):
1. Copies today.png to archive/YYYY-MM-DD.png (full day archive)
2. SCPs archive PNG to botpi:/var/www/html/mound/graphs/archive/
3. SCPs 7day.png to botpi:/var/www/html/mound/graphs/
   (7day.png generated by generate_graphs.py)

NOTE: If MOUND loses power before 23:30, the archive will not fire.
Use graph_day.py to manually regenerate missing archive entries.

================================================================================

generate_graphs.py
PURPOSE: Write a gnuplot script and run it to produce graph PNGs.

WHAT IT DOES:
1. Preprocesses the monthly data file
   - Strips comment lines
   - Merges date+time into one field (2026-06-01T00:00:02)
   - Filters today's data only
   - Writes clean version to /tmp/mound_clean.txt
2. Calculates dynamic y-ranges from actual min/max values in the data
   - 10% padding above and below
   - Filters out corrupt readings (< -100)
   - Gaps shown for lux overflow (99999)
3. Writes gnuplot script to generate_graphs.gp
4. Runs gnuplot to produce today.png
5. At 23:30 also generates 7day.png from last 7 days of data

GRAPH PANELS (6 stacked, shared time axis):
- Air Temperature    dynamic range
- Humidity           dynamic range
- Pressure           dynamic range
- Light              dynamic range (lux overflow shown as gap)
- Soil Temperature   dynamic range
- Soil Moisture      dynamic range

Rain events shown as blue vertical impulses on each panel.

DYNAMIC Y-RANGES: each panel auto-scales to the actual data range for
that day or week, making small variations clearly visible.

================================================================================

generate_page.py
PURPOSE: Build the HTML data page and latest.json from current data.

WHAT IT DOES:
1. Reads the last line of the monthly data file
2. Parses each field individually, each in its own try/except
   - Converts soil moisture raw value to descriptive text:
     < 13000 = Saturated
     < 15000 = Moist
     < 17000 = Moderate
     >= 17000 = Dry
   - Converts lux >= 99999 to "Overload"
3. Builds archive link list from local archive directory
4. Writes latest.json to ~/mound/data/latest.json
   (used by sensing.html live teaser via fetch())
5. Writes complete index.html to ~/mound/page/index.html
   (SCP'd to botpi by update_site.py)

PAGE STRUCTURE (mound/index.html):
- Nav + header (matches yesteryearforever site style)
- Current readings table
- Today's graph
- 7 day graph
- Latest camera image
- Daily archive links

================================================================================

camera.py
PURPOSE: Capture the daily noon photo and ship to botpi.

WHAT IT DOES:
1. At 12:30 only — captures one photo (1296x972)
2. Stamps timestamp onto image
3. Saves locally as ~/mound/images/noon/YYYY-MM-DD.jpg
4. SCPs to botpi:/var/www/html/mound/images/noon/YYYY-MM-DD.jpg
5. Also SCPs to botpi:/var/www/html/mound/images/latest.jpg
6. Errors logged to errors.txt

NOTE: Camera fires once per day only to conserve battery power.
The noon archive builds a daily timelapse over time.

TIMELAPSE ARCHIVE:
    botpi:/var/www/html/mound/images/noon/YYYY-MM-DD.jpg
    (one image per day, eventually assembled into seasonal timelapse)

================================================================================

graph_day.py
PURPOSE: Manually generate or regenerate a graph for any specific date.

USAGE:
    python ~/mound/scripts/graph_day.py 2026-06-08
    python ~/mound/scripts/graph_day.py          (defaults to yesterday)

WHAT IT DOES:
1. Reads raw data for the specified date from the monthly data file
2. Calculates dynamic ranges from that day's actual data
3. Generates a full day graph PNG
4. Saves to ~/mound/graphs/archive/YYYY-MM-DD.png
5. SCPs to botpi:/var/www/html/mound/graphs/archive/

USE WHEN: archive didn't fire automatically (MOUND lost power before 23:30)

================================================================================

generate_7day.py
PURPOSE: Force-regenerate the 7 day graph at any time.

USAGE:
    python ~/mound/scripts/generate_7day.py

WHAT IT DOES:
1. Collects last 7 days of data from monthly data files
2. Calculates dynamic ranges across all 7 days
3. Generates 7day.png with date-formatted x-axis
4. SCPs to botpi:/var/www/html/mound/graphs/7day.png

USE WHEN: 7day.png is stale or needs refreshing outside of the 23:30 cycle.

ERROR HANDLING
All errors logged to ~/mound/data/errors.txt with timestamp.
Check it with: cat ~/mound/data/errors.txt

Also contains DEBUG lines every 30 min showing hour/minute — useful for
confirming the script is running and diagnosing timing issues.

Common errors:
- ERROR TSL2591: Overflow  — too much direct sunlight, sensor maxed out
- ERROR BME280             — sensor wiring issue or loose connection
- ERROR camera SCP         — botpi unreachable or SSH issue
- ERROR DS18B20            — 1-wire device not found, check wiring
- ERROR: SCP failed        — network issue uploading to botpi

USEFUL COMMANDS

Check latest readings:
    tail -5 ~/mound/data/2026-06.txt

Check voltage trend:
    tail -20 ~/mound/data/2026-06.txt | awk -F', ' '{print $1, $9}'

Check error/debug log:
    cat ~/mound/data/errors.txt
    grep "DEBUG" ~/mound/data/errors.txt | tail -10

Run scripts manually:
    python ~/mound/scripts/log_sensors.py
    python ~/mound/scripts/update_site.py
    python ~/mound/scripts/camera.py

Regenerate specific day graph:
    python ~/mound/scripts/graph_day.py 2026-06-15

Force 7 day graph update:
    python ~/mound/scripts/generate_7day.py

Take a test photo:
    rpicam-still --vflip --hflip -o ~/mound/test.jpg

Check WiFi signal:
    iwconfig wlan0

Check I2C devices:
    sudo i2cdetect -y 1

Check crontab:
    crontab -l

Check cron logs:
    journalctl -u cron --since "1 hour ago" | tail -30
